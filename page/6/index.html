最大子段和

题目链接
很基础的题，遍历扫一遍记录前缀和，如果当前小于0则赋值为0，然后不断更新最大值

环状最大子段和

如果最优解的区间不经过$a_n$到$a_1$,那就是上面那个题，如果经过了，那么最优解的补集区间就是最小子段和，用总的和减去其即可
==注意最小子段和不能为全部序列，所以1到n-1和2到n跑两次==
（当然要根据具体题目判断是否原序列可以为空）
所以分类讨论
先找出不经过边界的最优解，再找出最小子段和，用总和减去它
这两个方案取最优解就可以了

双子序列最大和

点这里
维护一个前缀最大子段和和后缀的最大子段和，然后枚举分界点来更新答案就行
注意这题两个子段不能相接，所以我们更新以及枚举边界时要注意枚举的是分割点，然后取左右的答案(同时考虑能相接 那就是最大子段和)

环状最大双子段和

还是按照分类讨论的思想，假设有一个子段经过了$a_n$到$a_1$，那就是找出两个最小双子段，总和减去它，如果不经过，那就是上面那个题，两者取最优解就可以了
注意，两个最小段相连和两小段某一个为空和全为空都不会出现bug,因为第一个情况就把这些全考虑进去了
==注意两个子段不能覆盖全部的区间，所以应该从2到n-1==

线段树 & 分块

线段树

基础算法中的高级算法 高级算法中的入门算法
线段树是各种高级数据结构的核心，可以以一个优秀的复杂度维护区间内任意可合并的信息
单次修改和查询的复杂度一般在 $ O(logn) $ 左右 数据规模一般不超过 $ 2e6 $
常用宏

以维护区间最大值为例

线段树的写法是多变的，根据不同题目应当设计出不同的维护方式，一般来说主要是push_up和push_down这两个函数的设计，关于懒标记lazy tag应当是基本功，需要掌握

分块

属于数据结构中的毒瘤
以单次修改查询大致为$ sqrt(n) $ 的复杂度 从而使数据规模在 $ 2e5 $ 及以下时，能实现各种骚操作 个人很喜爱这个数据结构
每次修改和查询都是 大块直接标记，小块暴力处理
最差的情况就是 $ sqrt(n) $ 个大块和 $ 2 * sqrt(n) $ 个零散块 所以单次复杂度就是 $ sqrt(n) $
基本变量与宏

当你需要查询一个区间大于某个权值的数的个数时, 你可以分块

强烈建议板刷hzw专题来学习分块

偏序关系 & CDQ分治

二维偏序

逆序对问题
对于一个长度为 $ n $ 的序列，找出所有的点对 $(i, j) $ 满足 $(i < j)$ 同时 $(a_i > a_j)$ 的点对的数目
solution :

很容易想到，对于每个数的贡献，就是它前面大于他的数的个数，那么我们可以采用权值线段树的思想，不断查询大于他的权值的区间的和，不断更新来算贡献，对于这种单点查询，区间更新，我们考虑树状数组, 按照下标从小到大遍历，每个点的贡献就是 query(n) - query(a[i]) 然后对于每个点，将这个点的权值 $a_i$ 存入树状数组中

一道变式题
仔细想想，这其实是个二维偏序，对于每个位置的数有两种值，一个是$ L $ 一个是 $R $，那就是对于每个位置前面找出$ L $大于这个位置的 $R$的点对的数目，但是更新这个点权值到树状数组时应当用 $ L $来更新，可以利用 $ map $或者离散化后权值桶来处理出 $ L $ 和$ R $数组，之后直接跑一个逆序对就可以了

三维偏序

陌上花开
这次增加到了三维，经典模板题陌上花开

![](E:\iyua\source_posts\偏序关系-CDQ分治\屏幕截图 2020-12-22 230744.png)

我们就不能采用上述方法做了，$ IOI $ 选手 $ cdq $ 率先引入了 $ CDQ $分治这一思想来解决这些问题，我们先按照第一关键字排序，然后不断递归分治，从小区间合并为大区间，把左区间的贡献不断累积到右区间上面，对于递归过程中，我们能保证左区间相对于右区间第一关键字一定符合要求，之后二三关键字就优化成了二维偏序，一般采用树状数组来实现

一个变式
例如动态逆序对

![](E:\iyua\source_posts\偏序关系-CDQ分治\屏幕截图 2020-12-22 233025.png)

由题意可知 如果不带修改的话 那就是个裸的二维偏序逆序对问题，但是加入了修改操作，就不好做了，我们考虑三维偏序给每一个修改操作都增加一个时间轴，初始的序列时间轴为1，每次修改操作时间轴依次加一，那么这个题就转成了$ (i, v, t) $ 这样的三元组 求偏序， 因为 对于每次修改， 所减少的逆序对数就是 时间戳小于等于它的，权值大于它且在它前面 和 权值小于它且在它后面 的数的个数 那我们算出每个数的逆序对数， 把初始序列的逆序对都加起来， 再依次减去每个时间轴所影响的逆序对，就是每次的答案

再比如一道[牛客题](Why Did the Cow Cross the Road III (P))

大意 给出两个长度同为 $n$ 的排列， 第一个排列中元素值为 $v$的牛要走到第二个排列中元素值为 $v$ 的位置，一个冲突是指 两头牛的路线会交叉 且俩者的权值之差大于 $k$ 求问会产生多少冲突

仔细想想就会发现这也是个三维偏序问题，对于一个权值 $v$， 在第一个排列中下标为 $a$，在第二个排列种下标为 $b$，那麽就是一个三元组 $(a, b, v)$， 求问有多少个逆序对， 这里逆序对被定义为 $a_i < a_j$, $b_i > b_j$，且 $abs(v_i - v_j)$ > $k$

那就直接上 $CDQ$ 分治咯

CDQ分治毒瘤题

这题 $debug$ 直接 $de$ 到脑溢血了

考虑最近点位于它的左下方，那么 $|a_x - b_x| + |a_y - b_y|$ 可以化为 $(a_x + a_y) - (b_x + b_y)$ , 同时给每个点编上时间戳，那么就是(x, y, t)这个三元组 满足 $x_i <= x_j, y_i <= y_j, t_i <= t_j$ 的最大的 $x_j + y_j$ ，就是最小距离， 考虑旋转四次90°使得每个方向都会被考虑到，同时剪枝剪掉不在此范围的，且整体偏移一格去消0的影响（不好用树状数组维护0)

四维偏序

CDQ 套 CDQ
有两种写法，一种是归并排序，标记 $flag$ 判断在第一维时的左右区间， 另一种是直接 $sort$ 多传一个参数，来限制套用的 $CDQ$ 时计算的贡献
第一种写法常数会小一些

简单博弈

巴什博弈

这算是最简单的博弈论了
通常为 n个石子 两个人每次可以取1到m个石子
哪一方不能取则失败
结论是
n%(m+1)==0时先手必败 反之必胜
证明如下
当n%(m+1)不为0时 先手取 n%(m+1)个石子
之后只要一直控制石子是（m+1）的倍数 对方则无法获胜
因为 对方每次取得石子为1到m 假设取了x 那自己只需要取m+1-x则可以控制数量始终为m+1的倍数 所以取模不为0就先手必胜

然后就是放一个 稍微进阶点的博弈论题目吧

比如 wust oj 2305

我刚写的时候也懵了 只会套模板 不知道怎么输出步骤
但是我们想一想

在先手必胜的情况下 对于每一堆石子的总数的SG值 全部异或后不为0 我们的最优步骤就是 取走某一堆石子的某一部分石子 使得所有的SG值异或后变成0 使对手处于必败态那不就是我们的最优步骤吗
假设所有石堆的SG值异或结果为val 那如果我们用val再分别异或一下每堆石子的SG值 那根据异或的规律 我们求得的就是除了这一堆外 其他所有的异或值
即 ans^SG[arr[i]]就是其他石堆的异或结果
那假设我们取走cnt个石子 该堆剩下石子为num=arr[i]-cnt
如果SG[num]==(ans^SG[arr[i]]) 那么之后的异或就为0！！就处于必败态了

对于每一堆石子 只要取走一部分使剩下异或为0就可以

注意坑点
1 异或的优先级小于判等。。。就是说 a^b == c 和a^c==b是不一样的
写成（a^b)==c
2 记得输出方案时要比较SG值 不是比较石子数目！！

斐波那契博弈

有一堆物品，两人轮流取物品，先手最少取一个，至多无上限，但不能把物品取完，之后每次取的物品数不能超过上一次取的物品数的二倍且至少为一件，取走最后一件物品的人获胜。
先手胜当且仅当n不是斐波那契数（n为物品总数）

环形博弈

n个石子围成一个环,每次取一个或者取相邻的2个。
石子数目小于等于2 先手胜，其他 后手胜。

对称博弈

n个石子围成环,每次只能取相邻的1 - k个
如果k < n： 对k=1,如果n能被2整除,则后手赢；如果k>1,后手赢（先手取什么位置后手就取对称的位置，这样保证后手永远能取到)
如果k>=n： 先手赢.

C. Sereja and Brackets
长度 $ 1e6 $ 的括号序列，每次查询询问一个区间合法的匹配数
考虑线段树维护区间合并，每个节点维护三个值 此区间多余的左括号 此区间多余的右括号 此区间已匹配的括号数
那么合并区间就是 左匹配 + 右匹配 + min(左多余左, 右多余右), 同时记得用新产生的匹配数去更新左右多余的括号数目
对于查询操作，query操作应当写成返回值为$ node $类型，方便处理不在同一个小区间的匹配的情况
D. Powerful array
给出长度 $ 2e5 $ 的序列， 每次询问求一个区间内 每个数乘以这个数出现的次数 的和为多少
莫队入门题，直接上分块就可以了，注意 add 和 del 函数的书写
B. Modulo Sum
长度为 $ n $ 的序列，求问是否有一个子段的和是 $ m $的倍数
为什么混入一个DP..
首先根据鸽笼定理， $ n > m $ 必定可以， 小于的情况直接DP就可以了
C. Longest Regular Bracket Sequence
给出一个括号序列，求问最长连续成功匹配的长度是多少，并输出方案数…注意特判无解的情况
很简单，先扫一遍给每个点标记下能否成功匹配，然后用类似于最大子段和的思想就可以啦
D. Zigzags
给出长度为$ 3e3 $的序列，求问 满足 $ (i < j < k < l) $ 且 $ a[i] = a[k], a[j] = a[l] $ 这样的组合的数目
我按照逆序对的思想做的，对于每个点，找到它后面且权值与它相等的点，暴力连成一个双元组，那么这个题就转换成了对于一个双元组(a, b)，满足 $ a_i < b_i $, $ b_i < a_j < b_j $ 这样的双元组的数目，考虑逆序对的思想，用树状数组维护，但是有 (1, 4),(1, 5) 这样特殊的情况，所以我们不能靠常规的add函数来维护树状数组，我们应当保证当前的$a_j$严格大于前面的$a_i$时，才能 放入树状数组内，所以加个小技巧考虑下这种case 就可以了
D. Yet Another Yet Another Task
给出一个长度为 $ 1e5 $ 的序列， 找出一个子区间使得，这个子区间的和减去这个子区间最大值 所得到的结果最大，每个数权值$ a_i $ 满足 $ -30 <= a_i <= 30 $
考虑最大子段和的思想，由于这题权值范围很小，可以暴力枚举最大值，然后寻找这个最大值范围内的最大子段和
D. Pashmak and Parmida’s problem
简单逆序对，给出一种权值定义方式，然后找出逆序对的个数
先预处理出每个数左边和右边有多少个数与它相等(可以用离散化来求，map会不会T不清楚,然后建立一个树状数组就可以了
E2. Three Blocks Palindrome (hard version)
给出长度为$ 2e5 $ 的序列，然后定义了三段回文的含义， 求最长回文长度
由于权值小于200,所以我们可以存下每个权值存在的所有下标，然后暴力枚举两边的权值以及长度，对于中间的长度， 暴力枚举权值并用二分确认个数（注意清空时的复杂度)
D. Multiset
给出一个长度为 $ 1e6 $的序列，之后尽情若干次操作，要么插入一个数，要么删除第 $k$ 大的数，
插入一个数考虑权值线段树， 第$k$大考虑主席树，但是这题空间限制很紧，所以采用树状数组实现权值线段树，同时由于是整体第$k$大，不是区间第$k$大，所以可以用二分加树状数组来实现
D. Good Substrings
长度为1500的字符串，给出好串的定义，求问多少个本质不同的好串且子串
用前缀和的思想可以 $O(1)$ 判断是否为好串，关于去重，由于长度不大，可以采用哈希+set或者用字典树来去重
D. Vasiliy’s Multiset
插入，删除，查询异或最大
01字典树板子题…
B. Little Elephant and Array
长度1e5的序列，每次查询一个区间满足权值等于他出现个数的数的个数
考虑莫队，维护一下当前区间每个数出现的个数就可以了，且不需要离散化，因为大于n的数一定不满足条件（就算全是序列内全是这个数，个数也不够）直接跳过就可以了
B. Mike and Feet
对一个长度 $ 2e5 $的序列，找出1到n所有长度的子序列的最小值最大为多少
暴力，考虑单调栈优化，找出每个数左边和右边第一个小于它的数的地方，假设为$l$和$r$，那么这个点的贡献就是所有不超过$r-l-1$这个长度的答案都可以用目前这个点的大小来更新一下最大值，建立个权值线段树维护区间最大值就可以了
E. Enemy is weak
给出长度为$ 1e6 $的序列，找出所有的$(i < j < k)$, 且$( a_i > a_j > a_k)$，这样的组合数目
看给了5s，企图分块被卡了 预处理出每个数左边多少个数大于他，右边多少个小于它，然后乘起来就是它的贡献，累和起来就可以了，预处理的方法就是用逆序对，树状数组
E. Lomsat gelral
找出每个节点，它子树中出现次数最多的颜色乘于它的次数
树上启发式板子题，注意树剖的写法，以及容斥时避开重儿子
D. A and B and Interesting Substrings
给出一个$1e5$字符串，找出满足起始和结尾相等，且中间权值之和为0的子串的数目
先预处理出前缀和，然后记录下每个字母出现在哪些下标中，对于同种字母，就是找出两者sum差值为这个字母的权值的点对的个数，可以用 $map$ 来记录
