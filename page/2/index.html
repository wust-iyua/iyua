割点,桥,双连通

核心数组
$dfn$ $low$ 分别代表时间戳和最早能回溯到的时间戳
边双和点双针对无向图，其他的有向图上也可以写

割点

对于一个联通图，去掉某个点后，原本连通的两个点变得不再连通，则这个点就是割点
对于根而言，只要儿子数大于1就是割点
对于非根，利用上述两个数组的性质得知，如果一个点 $v$ 在另一个点 $u$ 的搜索子树内所能回溯到最小时间戳大于等于$u$点，那就代表 $u$ 把$v$点和外面的点割开了
给出无向图求割点

如果让我们计算，删去一个点后，所形成的新连通分量的个数，只需额外加个数组来记录
如POJ1523
这里处理出的 $num$ 数组是每个点的子树个数，那么 原本的联通图个数 $res$ 加上这个点的子树个数 $num[i]$ 减去一就是答案了

  • 注意根节点的子树要减去1
    可以理解为 原本 $res$ 个联通块，删去 $u$ 点，就先把 $u$ 的那个块删掉变成 $res - 1$ 之后把 $u$ 的子树个数加进去，所以答案是 $res + num[u] - 1$

单源最短路

Dijkstra

基于贪心的思想，每次寻找一个目前最近的点，然后用这个点松弛其他点
不能处理负权，因为是基于贪心寻找此时最近的点，如果存在负权边，则不能保证正确性，算法复杂度在于每次找最近的点需要 $O(n)$ 遍历所以点，加上要循环 $n$ 次，所以是 $O(n^2)$, 使用小根堆可以使查询最近点复杂度降为 $log(n)$， 所以 $heap$ 优化的 $dijkstra$ 复杂度为$O(nlogn)$

BellmanFord

以边来迭代松弛，可以处理负权，可以判断负环，由于每个点到起点的最短路一定最多经过$n-1$ 条边，所以只需迭代$n-1$次, 如果迭代 $n-1$ 轮后还能继续松弛，则代表有负环
由于迭代 $n-1$ 次，每次松弛所有边，所以复杂度为 $O(nm)$

SPFA

队列优化版的$BellmanFord$， 在极端情况下会被卡的和 $BellmanFord$ 一个复杂度，所以能不用就不用
可以处理负权，判负环，记录每个点入队次数，如果入队次数超过 n - 1 次，则代表有负环
注意这里如果有和起点不连通的负环，是无法判出来的
所以可以选择多次 $spfa$ 遍历所有联通块
$ O(玄学) $ 大概在 $O(km)$ k一般是个常数，可能会被卡到n，所以极端是 $O(nm)$

多源最短路

Floyd

可以处理负权，判断负环
处理出所有节点之间的最短路
注意去掉自环（如果题目说自环也算负环，那就在输入时特判）

JohnSon

针对稀疏图的多源最短路算法
考虑多源最短路优化 对每个点跑一次 $Dijkstra$ 来 $O(n^2logn)$ 求出解，但是如果含有负权就不好使了，（给每个边加一个固定权值使得全部为正数是错误的，因为松弛时

真正的距离应该是dis - 边数乘与这个正数

可以看到，加上的权值对两者影响不一致，所以错误
考虑如何使加上权值后，对于两点来说权值影响是一致的，给出一篇不错的博客
这里我们选择先进行一次 $BellmanFord$求出所有的点到 $1$ 的距离，然后给每条$(u, v)$边，加上权值$dis[u] - dis[v]$，之后就可以跑 $Dijkstra$了
给出证明
不含负权边
考虑最短路的性质，对于$(u, v)$ 这条边，显然 $dis[v] <= dis[u] + w$， 所以对于原有的权值 $w$ 加上 $dis[u] - dis[v]$ 得到的就是 $w + dis[u] - dis[v]$ ，根据上述最短路性质式子移项之后显然得证新边必定不是负数
**对两者影响一致**
考虑松弛 $dis[v] > dis[u] + w$ 这条边
假设左边此时$dis$值代表的从起点到v的最短路一系列点为 x1 x2 x3…..xv
那么权值就是 w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-dv
假设右边此时$dis$值代表的从起点到u的最短路一系列点为x1 x2 x3…..xu
那么权值就是 w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-du
此时这条边w就是 wu + du - dv
那么代入式子
$$
w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-dv > w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-du + w_u + du - dv
$$
化简可得
$$
w_{sum1} + d1 - dv > w_sum{sum2} + d1 - dv + w_u
$$
显然对两边的影响都是 $d1-dv$ 是一样的，所以正确性有保障
证毕

Dijkstra复活啦！可以处理负权了!
并不是，这个要事先跑一个 $BellmanFord$ 处理一次权值，如果是求单源最短路的话，负权还不如直接$BellmanFord$, 复杂度为 $O(nm) + O(n^2logn)$， 复杂度在稀疏图的情况下可以接受，所以这个算法只适用于稀疏图求多源最短路

虚拟源点

有时对于多起点，或者多终点的情况，需要建立虚拟源点和虚拟汇点，源点与所有起点相连，汇点与所有终点相连，然后以虚拟源点作为起点跑最短路就可以了，例如hdu2066

分层图

建立多层图，来用空间优化掉复杂的处理过程，给出几道例题
洛谷P4568
建立 $k$ 层图，对于输入的每条边 $w(u, v)$ 假设此时在第 $i$ 层，那就建立四条边

  • 第i层 u-> 第i层v w
  • 第i层 v-> 第i层u w
  • 第i层 u -> 第i+1层v 0
  • 第i层 v -> 第i+1层u 0

这样层与层之间的连边就等效于一次免费旅行,之后把第0层的 $s$ 当作起点，然后每一层的 $t$ 取 $min$ 就是答案了,不直接选取最后一层的原因是可能不必穿越 $k$ 次就是最优解
洛谷P1266
对于速度分层。设 dis[u][v] 代表以$u$速度到达$v$这个位置的最短路就可以了，最后终点的每种速度取最小值
洛谷P5304
不能直接连接每个点和虚拟源点虚拟汇点，因为这样可能会导致起点和终点一致，导致错误
考虑起点和终点一定不一样的分组，我们分成两堆，一堆连接起点，一堆连接汇点就可以了，那么如何分组呢，假设最优解是 $(u, v)$ 且 $u$ 不等于 $v$ ，那么这两个点二进制表示法一定有一位不一样，所以考虑枚举二进制进行分组，这样必定可以得到最优解
但由于是有向图，所以对于每次分组跑两次，一次为0的连接起点，一次为1的连接起点
CF div2 702 E
对于点进行分类，第一种是奇数步数到达这里的，第二种是偶数步数到达这里的，那么分类讨论，如果是奇数，那代表这个点应该是每次跨两步的第一步，偶数就是跨两步的第二步，所以我们同时记录这个点目前是从哪条边跨过来的，第一类点直接记录为0就行，然后松弛时，对于第一类点，应该是不修改权值，加上目前跨的这条边变成第二类点进队，第二类点是修改权值，清空它所携带的信息成为第一类点

拓扑排序

一个在 $dp$ 与 图论领域十分实用的一个算法，流程十分简单

  1. 找到所有入度为0的点，放入队列
  2. 每次取出一个度数为0的点，之后把他所有的后继度数都减1，把新产生的度数为0的点放入队列

只要最终所有点的度数都为0，（或者入队点数等于总点数），那么代表这个图就是一个有向无环图，可以进行判环，也可以在拓扑排序过程中进行 dp 等一系列操作
如果需要求出拓扑序列字典序最小，只需要改成优先队列，优先弹出小的数就可以了
给出两道不错的题目
HDU4857
注意这里是1尽可能靠前，之后2尽可能靠前
并不是字典序！！如果直接上优先队列显然会出错
比如对于两条链
2 3 4 5
6 1
你用优先队列跑出来的是 2 3 4 5 6 1, 但实际上应该是 6 1 2 3 4 5最优
考虑字典序最小与1尽可能靠前的区别
如果你用优先队列，那么他会尽可能把大的数字放后面！来保证字典序最小
所以我们反向建图，然后优先队列先找大的，最后反向输出就可以了

比如说对于一组约束关系 $x - y >= w$
我们考虑最短路松弛的过程，对于一条边 $(u, v)$ 权值为 $w$， 显然我们可以得到 $dis[v] <= dis[u] + w$ ….（显然）
然后对上面那个式子进行变式为下面那个格式就是 $y <= x - w$ 所以等效于从 $x$ 到 $y$ 建立一条权值为 $-w$ 的有向边
最后跑出的每个最短路数组就是一组解

先放出一个性质（说实话这个我看不懂证明….背住就行！）

对于没要求最大最小差值的题目，可以最长最短路都可以求解
但是如果题目规定解的下限，则求最长路，规定了上限，就求最短路
如果最短路出现负环，最长路出现正环，都代表无解
窝真的看不懂这个证明哇
然后对于每个约束条件 $x - y >= w$ （注意这里不是严格大于）
考虑边的松弛方式
对于最短路 $w (u到 v) $ 显然 dis[v] <= dis[u] + w**
对于最长路 $w (u到v) $ 显然 **dis[v] >= dis[u] + w
之后只需对约束条件进行移项，使之与上述对应就可以啦
考虑约束条件的变式

  1. x - y > w 严格大于，只需对w进行小的修改就可以了，int就是 x - y >= w + 1 , 实数就是 x - y >= w + eps 严格小于类比这个，同理
  2. x == y 这种情况只需改为 x - y >= 0 和 x - y <= 0 这两种情况就可以了
  3. x == w 规定了 初始权值，只需建立虚拟源点S 然后 x - S >= w 和 x - S <= w这两条边就可以了

由于约束条件很容易出现负数权值，所以一般不能用 $Dijkstra$ 算法，然后数据范围允许的话尽量用 $BellmanFord$ 来求解，复杂度稳定且易于书写，尽量少用去世算法 $spfa$， 如果保证是非负权值，则上迪杰斯特拉就可以了

注意事项
  1. 如果建立虚拟源点了，记住 $spfa$ 和 $bellmanford$ 判环时松弛次数不是 $n - 1 $ 而是 $n$..$spfa$ 和 $BellmanFord$ 都是最多松弛 $n - 1$ (点数减一)轮，超过这个限制就是有环
  2. 对于最长路，可将权值取反跑最短路来方便求出
  3. 特判掉不合法数据，比如规定 $x - y >= w$ 时，$x$ 与 $y$ 相等的情况

树的直径

对于边权值为$1$ 的情况，可以通过两次 $BFS$ 或者 $DFS$ 或者树型 $DP$ 来求解
$dp$ 写法可以处理出权值为负的情况
dfs和bfs
很简单，先随便找一个点进行搜索，找出离这个点距离最远的点，之后再以那个点再搜索一次，两次搜索所得到两点就是直径的两个端点

此时 $mx$ 为直径长度，$u1$ 和 $u2$ 为直径端点
树型dp
这个只需要在每个点维护一个距离它不同子树内叶子距离最大的两个值，相加就是直径的长度，如果需要求出具体端点，则还需额外记录每个权值对应的叶子的下标

$ans$ 为直径长度

splay入门例题

二叉搜索树很有用，但是极端数据会把它卡到退化成一条链，所以需要更高级的平衡树， $splay$ 就是对于每次操作的点都进行一次旋转来保证玄学复杂度

关于 $splay$ 维护序列的好处是按照下标建立 $splay$ 中序遍历结果就是当前序列，序列中第 $k$ 个数就是 $splay$ 的 $kth$ 位置，按照这个思想来维护就可以了，同时由于旋转不会改变中序遍历结果，所以也不会改变序列结构

$splay$ 处理区间反转经典题，首先我们知道不论如何反转，对于一颗平衡树而言，中序遍历的结果一定是不变的，同时交换左右子树，序列就会等效于反转
所以我们对下标来建立平衡树，之后对于每次 $l, r$ 区间就是把 $l+1$ 旋转到根， $r+1$ 旋转到根的右儿子，这样就把 $l, r$ 这个区间分割出来了，直接打上一个标记记录这个区间需要反转，同时在查询和输出时应当下放标记，利用线段树 $lazytag$ 的思想就可以简单的实现

01序列处理sort(线段树)

线段树妙用
CF558E
显然我们有个 暴力写法每次都 $sort$ 一下
考虑线段树，开26棵权值线段树维护每个字母在哪些下标出现，（显然要动态开点），最终输出答案就遍历所有线段树赋值给每个位置就可以了
现在问题是如何使用线段树来维护每次的 $sort$, 考虑区间覆盖，起初每个叶子为 $1$ 代表整个位置有一个字母，为 $0$ 则表示没有，对于一次 $sort$, 我们从 $a$ 遍历到 $z$, 逆序只需反着来就可以了，本质是一样的，所以我们只讨论顺序怎么排
对于一次 $sort$ 从 $l$ 到 $r$ ，我们肯定是先把小的字母放前面（即尽可能靠近 $l$ ），所以我们记录一个 $pos$ 代表此时这个字母应该从那个位置开始往后铺，初始为 $l$, 然后处理每个字母时，先查询这个字母在 $l, r$ 这个区间出现了多少次，设为 $cnt$ 那么显然我们需要将 $(pos, pos + cnt - 1)$ 这个区间都置为这个字母，同时消除 $(l, pos - 1)$ 和 $(pos + cnt, r)$ 区间的字母，然后处理下个字母时需要更新 $pos$ 为 $pos + cnt$ ，对于赋值和消除就是区间置为$0$和 $1$ 的区别
这样我们就愉快的解决了这道题啦
