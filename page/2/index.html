传送门
如果没有乘与 $x$ 这个操作，那就是一个简单的贪心，但是加上这个操作后直接贪心就会出错
如果我们找出一段序列乘与 $x$ 后，设被乘与的序列为1，其他的序列为0，那么最后的序列必定是一个类似于001100这样的三段序列，且每一段可能为空
考虑 $dp$, 设置 $dp[maxn][3]$
$dp[i][0]$ 表示第 $i$ 个位置以及它前面的位置没有被乘与 $x$ 的最大子段和
$dp[i][1]$ 表示第 $i$ 个位置被乘与了$x$ 的最大子段和
$dp[i][2]$ 表示第 $i$ 个位置以及它后面都不会被乘与 $x$， 但它前面被乘与 $x$ 的最大子段和
那么我们显然可以得出状态转移方程

AC自动机

由于不少题目都是数据结构结合字符串一起出的，所以就去简略学了一下 $ac$ 自动机
关于 $ac$ 自动机，共有两类题目

  1. 结合数据结构跑字符串匹配
  2. 结合矩阵快速幂求解 $DP$

我 $dp$ 实在是太不行了，只写了一个 $ac$ 自动机求解 $dp$ 的例题…（但是我队友会）

所以本文基本都是在说第一类应用

前置知识当然是 $Trie$ 树，然后强烈建议先学会 $kmp$ ,两者本质十分类似，$ac$ 自动机就是在树上跑 $kmp$ 要求是灵活应用 $Trie$ 树，毕竟变化多端，常见技巧如记录当次下标，沿路径赋值，终点建立 $vector$ 等都要会

一个基础但重要的结论
$Trie$ 图上每个结点都代表着一个字符串，且每个节点的 $fail$ 指向的节点都是一个和它拥有最长公共后缀的字符串

考虑字符串多模式匹配，例如给出一系列文本串，再给出一个模式串，求问多少个文本串在模式串中出现过或出现过多少次
显然我们有个对每个文本串都跑个 $kmp$ 的暴力做法，但是复杂度显然会被卡到 $O(n^2)$
这时候就用到 $ac$ 自动机了，它首先把所有的文本串都放在 $Trie$ 树上面，之后拿模式串在树上同时匹配所有的文本串

我们来考虑 $ac$ 自动机匹配的过程，显然一个结论是一个字符串的每个子串都是这个字符串的一个后缀的前缀（前缀的后缀），而 $ac$ 自动机匹配就是用的这个性质，在匹配时是取出这个字符串所有的前缀，然后用这个前缀在 $Trie$ 上匹配，不断跳 $fail$ 边，在跳 $fail$ 的过程就是快速枚举后缀的过程，那么显然如果模式串的某个前缀的后缀是文本串，那这个前缀在跳 $fail$ 的时候一定会经过文本串所代表的节点
同时考虑每个点只有一个 $fail$ 边，就是出度为1，我们把所有 $fail$ 边反建，那么每个点的入度就是 $1$, 那其实 这时候 $fail$ 形成的图就是一个有向有根树
然后我们如果跳 $fail$ 时会经过这个点，那就说明这个点一定是那个前缀的节点的一个祖先，那么求 $x$ 在 $y$ 出现了多少次这个问题就转换成了求 节点 $x$ 在 $fail$ 树上的子树中有多少个点是属于 $y$ 的（就是 $y$ 的所有前缀的节点哪些在 $x$ 那个节点的子树内）
所以我们可以得到如下算法，读入所有文本串建立 $Trie$ 树，同时记录每个字符串所代表的节点， 然后反连 $fail$ 建立 $fail$ 树，标记所有属于模式串的节点为1，之后就是子树求和了
我们可以利用 $dfs$ 序记录 $fail$ 树的时间戳，然后树状数组/线段树求区间和就可以了

二分图与匹配

定义

二分图是图论中一种特殊的模型，对于二分图的点集，我们一定可以把它分成左右两部分，使得左右两部分内部无任何连边，所有存在的边都是从左到右或者从右到左，显然二分图不含有奇环
交替路 从一个未匹配点出发，依次经过非匹配边，匹配边，非匹配边…这样形成的路径
增广路 交替路中除起点外又遇到一个未匹配点的路径

二分图判定

由于不含奇环，所以我们只需暴力染色判断是否有相连的点相同颜色就可以了

二分图最大匹配

左右两部分的点，一对一进行匹配，所能得到的最大匹配成功数目就是最大匹配数
常用技巧需要用到 染色，拆点等，多染色，少用些奇奇怪怪的东西，找不出错误就用染色法
有匈牙利算法和HK算法这两种求法，前者复杂度是 $O(n^3)$ 但是书写较易且容易理解，后者复杂度是 $O(sqrt(n) * m)$, 常用于大数据范围

二分图博弈

我直呼学到了！！！
考虑这样一种问题，对于两种状态，不同状态可以相互转换，相同状态不可以，（不能走走过的地方），谁不能转换谁就输了这种博弈
注意状态只有两种，且只有不同状态之间有连边，我们完全可以建立二分图模型考虑二分图最大匹配，我们分类讨论
完美匹配
每个点都有匹配点，那么显然不论先手选的是左/右部点，后手都一定可以走向与之匹配的右/左部点，这样无论怎样后手都有应对方案所以先手必输
非完美匹配
这种情况就会有孤点，先说结论，从孤点先走则先手必胜，因为如果孤点没有出度，那么后手直接输，如果有出度，则必定是对应一个匹配的左/右部点，按照上述完美匹配过程，这时无论后手如何走，先手都有下一状态可以走，（难道后手不会走到一个非匹配点使得断绝先手的下一状态吗？）并不会，你这种情况就是一条增广路，而最大匹配最终是不含有增广路的，所以不会出现这种状态，严格来说是从非必匹配点出发则先手必胜，因为考虑

这种连边，其实从1和3先手都是必胜的，1和3谁是孤点取决于你读入的顺序，但它们有一个共同点，他们都是最大匹配中非必匹配点，这里2明显是必匹配点，那如何来判断一个点是否为非必匹配点呢？
显然，没匹配成功的点一定是的，考虑已经匹配成功的，我们将其匹配拆散，然后对其原有的匹配点跑一次增广路，其中要特判避开这个点，如果成功增广则是非必匹配点，不能成功增广就是必匹配点，即删掉这个点后最大匹配数不变，则不是必匹配点

最小点覆盖

二分图中选取最少的点使得这些点和所有的边都有关系，就叫最小点覆盖
最小点覆盖等于最大匹配数

最大独立集

二分图中选取最多的点使得这些点之间互相没有连边，叫做最大独立集
最大独立集等于总点数减去最大匹配数

最小路径覆盖

在一个有向图中，选取最少的不相交的路径使得覆盖了所有点
最小路径覆盖等于总点数减去最大匹配数
如果要求路径可以相交，先跑一个闭包传递算法后就可以和上面一样处理了

基于二分图的最大团

最大团是最大完全子图，子图内任意两点都有直接的路径相交
这是个 $np$ 问题，但是对于特殊的图模型来说，是可以求得
我们知道最大独立点集是指这些点之间互相没有连边，刚好和最大团相反，那如果一张图的补图是一个二分图的话，我们对于补图求出的最大独立子集是互相没有边，而补图中的边代表原图没有边，补图中没有的边代表原图有边，所以补图的最大独立子集就是原图的最大团
但前提是他的补图一定要是个二分图的模型

何谓支配？指的就是给出一颗有向树，规定根之后，对于一个节点 $v$ ，如果从根到这个节点所有的路径都会经过点 $u$, 那就代表 $u$ 这个点 支配 了 $v$
而对于一颗支配树而言

  • 任意一个节点的所有祖先都支配了这个节点
  • 这个节点支配了它子树内的所有点

这样我们就可以在支配树上进行 $DFS$ 等一系列操作来解决实际问题了
如果是无向图的话，只需双向加有向边就可以啦，所以这里只讨论有向图
我们当然可以暴力枚举删去每个点来判断这个点支配了哪些点，显然复杂度不是很优秀，为 $O(n ^ 2)$
注意有时需要手动添加一个虚点，来连通整张图
考虑无环
那么就是 $DAG$，这种情况比较简单，只需动态维护 $LCA$ ，按照拓扑序来动态加点入新树中，每次一个节点的加入树里，他的父亲节点就是它原有的所有的父亲节点在新树上的 $LCA$ 这样构造出来的树就是支配树

路径

单源最短路

Dijkstra

基于贪心的思想，每次寻找一个目前最近的点，然后用这个点松弛其他点
不能处理负权，因为是基于贪心寻找此时最近的点，如果存在负权边，则不能保证正确性，算法复杂度在于每次找最近的点需要 $O(n)$ 遍历所以点，加上要循环 $n$ 次，所以是 $O(n^2)$, 使用小根堆可以使查询最近点复杂度降为 $log(n)$， 所以 $heap$ 优化的 $dijkstra$ 复杂度为$O(nlogn)$

BellmanFord

以边来迭代松弛，可以处理负权，可以判断负环，由于每个点到起点的最短路一定最多经过$n-1$ 条边，所以只需迭代$n-1$次, 如果迭代 $n-1$ 轮后还能继续松弛，则代表有负环
由于迭代 $n-1$ 次，每次松弛所有边，所以复杂度为 $O(nm)$

SPFA

队列优化版的$BellmanFord$， 在极端情况下会被卡的和 $BellmanFord$ 一个复杂度，所以能不用就不用
可以处理负权，判负环，记录每个点入队次数，如果入队次数超过 n - 1 次，则代表有负环
注意这里如果有和起点不连通的负环，是无法判出来的
所以可以选择多次 $spfa$ 遍历所有联通块
$ O(玄学) $ 大概在 $O(km)$ k一般是个常数，可能会被卡到n，所以极端是 $O(nm)$

多源最短路

Floyd

可以处理负权，判断负环
处理出所有节点之间的最短路
注意去掉自环（如果题目说自环也算负环，那就在输入时特判）

JohnSon

针对稀疏图的多源最短路算法
考虑多源最短路优化 对每个点跑一次 $Dijkstra$ 来 $O(n^2logn)$ 求出解，但是如果含有负权就不好使了，（给每个边加一个固定权值使得全部为正数是错误的，因为松弛时

真正的距离应该是dis - 边数乘与这个正数

可以看到，加上的权值对两者影响不一致，所以错误
考虑如何使加上权值后，对于两点来说权值影响是一致的，给出一篇不错的博客
这里我们选择先进行一次 $BellmanFord$求出所有的点到 $1$ 的距离，然后给每条$(u, v)$边，加上权值$dis[u] - dis[v]$，之后就可以跑 $Dijkstra$了
给出证明
不含负权边
考虑最短路的性质，对于$(u, v)$ 这条边，显然 $dis[v] <= dis[u] + w$， 所以对于原有的权值 $w$ 加上 $dis[u] - dis[v]$ 得到的就是 $w + dis[u] - dis[v]$ ，根据上述最短路性质式子移项之后显然得证新边必定不是负数
**对两者影响一致**
考虑松弛 $dis[v] > dis[u] + w$ 这条边
假设左边此时$dis$值代表的从起点到v的最短路一系列点为 x1 x2 x3…..xv
那么权值就是 w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-dv
假设右边此时$dis$值代表的从起点到u的最短路一系列点为x1 x2 x3…..xu
那么权值就是 w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-du
此时这条边w就是 wu + du - dv
那么代入式子
$$
w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-dv > w1 + d1 - d2 + w2 + d2 - d3 + w3 + d3 - d4……-du + w_u + du - dv
$$
化简可得
$$
w_{sum1} + d1 - dv > w_sum{sum2} + d1 - dv + w_u
$$
显然对两边的影响都是 $d1-dv$ 是一样的，所以正确性有保障
证毕

Dijkstra复活啦！可以处理负权了!
并不是，这个要事先跑一个 $BellmanFord$ 处理一次权值，如果是求单源最短路的话，负权还不如直接$BellmanFord$, 复杂度为 $O(nm) + O(n^2logn)$， 复杂度在稀疏图的情况下可以接受，所以这个算法只适用于稀疏图求多源最短路

有时对于多起点，或者多终点的情况，需要建立虚拟源点和虚拟汇点，源点与所有起点相连，汇点与所有终点相连，然后以虚拟源点作为起点跑最短路就可以了，例如hdu2066

分层图

建立多层图，来用空间优化掉复杂的处理过程，给出几道例题
洛谷P4568
建立 $k$ 层图，对于输入的每条边 $w(u, v)$ 假设此时在第 $i$ 层，那就建立四条边

  • 第i层 u-> 第i层v w
  • 第i层 v-> 第i层u w
  • 第i层 u -> 第i+1层v 0
  • 第i层 v -> 第i+1层u 0

这样层与层之间的连边就等效于一次免费旅行,之后把第0层的 $s$ 当作起点，然后每一层的 $t$ 取 $min$ 就是答案了,不直接选取最后一层的原因是可能不必穿越 $k$ 次就是最优解

但由于是有向图，所以对于每次分组跑两次，一次为0的连接起点，一次为1的连接起点

对于点进行分类，第一种是奇数步数到达这里的，第二种是偶数步数到达这里的，那么分类讨论，如果是奇数，那代表这个点应该是每次跨两步的第一步，偶数就是跨两步的第二步，所以我们同时记录这个点目前是从哪条边跨过来的，第一类点直接记录为0就行，然后松弛时，对于第一类点，应该是不修改权值，加上目前跨的这条边变成第二类点进队，第二类点是修改权值，清空它所携带的信息成为第一类点

但是对于有向图而言，如果需要求出所有点到起点的最短路，只需反向建边，之后得到的数组就是每个点到起点的最短路

bitset优化
对于求出联通性问题，不需要求出具体路径权值，可以用bitset优化这个过程
例如这道题牛客挑战赛47 C
我们建立两个图，一个装一定存在的边，另一个装可能存在的和一定存在的
所有边赋值为1代表可以走，为0代表不可走，之后 $bitset$ 优化跑最短路，对于询问，优先处理一定可走的就可以啦， 注意内外循环变量嵌套的位置！

最小环

显然，无向图一个环最起码要三个点，有向图最起码两个点
$sol1$
枚举每条边，然后删除掉，跑这条边两个端点之间的最短路，再加回来原有的边就是一个环，暴力更新最小环就可以了，复杂度十分感人….
$sol2$
跑 $Floyd$, 对于一个环，我们使其中标号最大的点为 $k$, 之后在 $Floyd$ 更新过程中，在松弛之前用这两条边加上最短路就是最小环
如果不是求全局最小环，而是求某个点上的最小环，那只需要枚举删去与这个点相连的边就可以啦
或者在 $Floyd$过程中只记录与这个点相关的最小环

那么很简单，离线逆操作，把所有没被删除边连起来，倒着处理询问，遇到删边就加边

在线呢？？
考虑对偶图

黑色是原矩阵，对偶图就是红色部分，可以看出对偶图就是把空白部分当成点，点当成空白部分，然后我们考虑如何判断
假如我们要断开绿色两点之间的边，那么如果此时两个紫色点已经联通，则断开后绿色点分成两个块，如果此时两个紫色点没有联通，则绿色点还是一个块
之后连接两个紫色点
但是如果这题询问和断边是分开的（就是先断边，然后询问任意两点联通性）…还强制在线就不会做了。好像是维护动态图联通性 我爬了
